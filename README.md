# マニュアル検索 (manual-finder)

手元のPDFマニュアルを**とっさに引く**ための、個人用・完全オフラインPWA。iPad 1台での運用を想定。
**外部ネットワーク通信ゼロ**（PDFのバイトを1ビットも外部に出さない設計）。

- ページ単位の全文検索（日本語bigram＋型番の前方一致）→ 該当ページを開いてキーワードをハイライト
- PDFごとにリッチメモ（本文の任意位置に写真を差し込める）・タグ・お気に入り
- 締切つき「施策（タスク）」を**月カレンダー上の期間帯**で管理（締切間近/超過を色で強調）
- 全データ（PDFバイト含む）を1ファイルにエクスポート／インポートで完全復元
- Service Worker で全アセットをプリキャッシュ → 機内モードでも全機能が動作

技術: Vite + React + TypeScript(strict) / pdfjs-dist / minisearch / dexie(IndexedDB) / fflate / vite-plugin-pwa。
すべてローカルにバンドル。CDN・Webフォント・解析・テレメトリは一切なし。設計判断の詳細は [`DECISIONS.md`](./DECISIONS.md)。

> **プライバシー**: PDF・メモ・写真などのデータは端末内（IndexedDB）にのみ保存され、どこにも送信されません。
> このリポジトリにはデータは含まれません（アプリの**コードのみ**）。

## セットアップ（開発・ビルド）

Windows + WSL2 / Mac不要。Node.js 20+ を推奨（開発は Node 24 で確認）。

```bash
npm install
npm run dev       # 開発サーバ（http://localhost:5173）
npm run build     # 本番ビルド → dist/
npm run preview   # ビルド結果をローカル配信（PWA/SWの動作確認）
```

- `dev`/`build` の前処理として `prepare-assets`（pdf.jsのcMap/標準フォントを `public/pdfjs/` へコピー＋アイコンPNG生成）が自動実行される。
- 生成物（`public/pdfjs/`・`public/icons/`）は `node_modules` から再生成されるため git 管理外。

### iPadへ配置する

このアプリは静的ファイル（`dist/`）だけで動く。iPadのSafariから開ける場所に `dist/` を置けばよい。

- 手軽なのは、同一Wi-Fi上のPC等で `dist/` をローカル配信し、そのURLをiPad Safariで開く方法。
  ```bash
  npm run build
  npm run preview -- --host   # 表示された http://<PCのIP>:4173 をiPadで開く
  ```
  外部公開は不要。ローカルネットワーク内で完結できる。
- サブパス配信でも動くよう、pdf.jsのアセットパスは実行時に `BASE_URL` から解決している（ルート配信が最も簡単）。

> **⚠ オフライン化・暗号化には「安全な接続」が必須**
> Service Worker（オフライン動作）と WebCrypto（バックアップ暗号化）は、ブラウザの仕様上
> **セキュアコンテキスト（`https://` か `http://localhost`）でのみ有効**です。
> `http://<PCのIP>` のような平文HTTPのLAN配信では、**オフライン化も暗号化も無効**になります（アプリ自体は開けますがPWAになりません）。
> iPadで完全オフライン＋暗号化まで使うには、次のいずれかで**HTTPS配信**してください:
> - 静的ホスティング（HTTPS）に `dist/` を置く（アプリのコードのみが配信され、**マニュアル本体は端末外に出ません**）。
> - もしくはローカルHTTPSサーバ＋iPadに信頼させた証明書。
> どちらの場合も、外に出るのは**アプリのコードだけ**で、PDF・メモ・写真は端末内にのみ保存されます。

## iPadへのインストール（ホーム画面に追加）

1. iPad Safari で本アプリのURLを開く。
2. 共有ボタン（□に↑）→ **「ホーム画面に追加」** をタップ。
3. 追加された「マニュアル」アイコンから起動（全画面・standalone表示）。
4. 初回起動時、ブラウザにストレージ永続化を要求する（許可されると保存が保護されやすい）。

> インストール後は機内モードでも動作する。初回に一度オンラインで開き、Service Workerのキャッシュを完了させておくこと。

## 使い方

### PDFの取り込み
- 「ライブラリ」タブ →「＋ PDFを取り込む」→ iPadのファイルからPDFを選択（複数可）。
- 取り込み時に本文テキストをページ単位で抽出し、検索インデックスに登録する。
- スキャン画像主体（テキストレイヤ無し）のPDFは、取り込み時の確認から**OCR（端末内文字認識）**を実行すると本文検索できる（詳細画面から後追い実行も可）。

### 検索（本アプリの核）
- 画面上部の検索バーにキーワードや型番を入力（インクリメンタル検索）。
- 結果は**ページ単位のヒットカード**（マニュアル名 + ページ番号 + 前後スニペット + 一致語ハイライト）。
- カードをタップ →「対象PDFを開く → 該当ページにジャンプ → 検索語をハイライト」。
- コツ: 型番は途中まで入力しても前方一致でヒット（例「RAS-X2」）。日本語はやや短めのキーワードが当たりやすい。

### メモ（リッチ）・タグ・お気に入り
- 一覧の各PDFの「⋯」から詳細を開き、タイトル/タグ/メモを編集できる。
- メモは**本文の任意位置に写真を差し込めるリッチメモ**（文章→写真→文章の縦混在）。写真はカメラ/フォトから追加し、順序ごと保存・バックアップされる。写真は🔎でOCRして検索対象にもできる。
- ライブラリはカテゴリごとの折りたたみ表示（規定は閉。開閉状態は記憶され次回復元）。
- ★でお気に入り登録。お気に入りは一覧最上部に大きめタイルで表示され、一瞬で到達できる。

### 施策（月カレンダー）
- 「施策」タブは**月カレンダー**。各施策が開始日〜締切日の**期間帯（タイトル付きバー）**で表示され、同時期の施策はレーンで積まれ、週をまたぐ帯は折り返す。
- 締切超過=赤、締切間近(3日以内)=橙、通常=青。前月/翌月/今日で移動。
- 帯をタップ → 詳細シート（編集・削除・**紐付けPDFをワンタップで開く**）。
- 登録項目: 施策名 / 開始日（帯の起点。未入力は締切日と同じ扱い） / 締切日 / メモ / 紐付けPDF(1つ・任意)。

### 共有（選んだPDFだけを同僚に渡す）
- ライブラリの「共有」→ PDFを選択 →「書き出す」で、**選んだ分だけ**のzipを作成。AirDrop等の**ローカル受け渡し**で同僚に共有できる（外部サーバー/クラウドは経由しない）。
- zipにはPDF本体・メモ（写真インライン含む）・ページメモ・検索用テキスト（OCR結果含む）が入る。**自分のカテゴリ/タグ/お気に入りは含めない**（受信側の分類体系を汚さない）。
- 受け取った側は「バックアップ」タブ →「ファイルを選んで取り込む」→「**追加で取り込む（マージ）**」。既存データは消えず、受け取ったPDFは「未分類」に**未読マーク付き**で入る（開くと既読）。
- 重複は「PDF本体もメモ・写真も完全一致」の場合だけスキップ。少しでも違えば両方残る（注釈が失われない）。
- ※ 渡した先の端末での取り扱いは各自の管理・責任範囲（アプリは通信では漏らさないが、共有そのものの可否は職場のルールに従うこと）。

### バックアップ（エクスポート／インポート）— 中核機能
- 「バックアップ」タブ →「⬇ エクスポート」で、全PDF・メモ・写真・施策・タグ/お気に入りを**1ファイル**に書き出し、iPadの「ファイル」に保存する。
- **パスワード暗号化（推奨）**: エクスポート時に「🔒 パスワードで暗号化する」を選ぶと、`AES-GCM`（鍵はパスワードから `PBKDF2` で導出）で暗号化した `.mfbackup` を出力する。パスワード無しでは中身を読めない。**パスワードを忘れると復元不可**なので厳重に保管すること。（暗号化はセキュアコンテキストでのみ有効。上記「安全な接続」を参照）
- 復元は「⬆ バックアップを選んで復元」→ ファイルを選択（**現在のデータは全置換で置き換わる**）。暗号化ファイルはパスワードを求められる。再インストール後・別端末でも完全復元できる。
- **運用推奨: 月1回程度エクスポートしてローカルに退避する。** iOSはストレージを退避することがあり、`navigator.storage.persist()` は保証ではないため、バックアップが最後の砦。
- 検索が効かない/結果がおかしいときは「↻ 検索索引を再構築」（保存済みPDFのテキストから作り直す。PDF再取り込みは不要）。

## 既知の制約

- **iOSのストレージ退避**: ホーム画面PWAでも、長期間未使用などでIndexedDBが退避される可能性がある。→ 定期的なエクスポートで対処。
- **容量の上限**: アプリ側に固定の上限は設けていない。実際に保存できる量は **iOS Safari が端末ごとに割り当てる容量（端末依存・可変）** に従う。画面上部に現在の使用量/割当量をリアルタイム表示し、割当量の約8割で警告する。設計上の想定は「現場で引く約100枚（数百MB）」だが、これは上限ではなく快適に使うための目安。全マニュアル一式（数GB〜）をまるごと入れる運用は想定していない（動作が重くなり、iOSの退避リスクも上がるため）。
- **スキャンPDF・写真**: テキストレイヤの無いPDFや写真は、**OCR（端末内文字認識）**で検索対象にできる（取り込み時に確認、または詳細画面から実行）。OCRは Tesseract.js を用い、認識用データも含めすべてローカル同梱で**外部送信ゼロ**。認識結果は保存され再スキャン不要。認識精度は完璧ではなく、1ページ数秒かかる。
- **日本語検索の精度**: bigram方式のため助詞ノイズ等で再現率は完全ではない。型番・品番・固有名の検索は実用十分。
- **PDFビューア**: 規定は**連続スクロール表示**（設定で「タップ移動＝1ページずつ」に変更可）。画面タップで上下バーの表示/非表示を切替。ページジャンプ・ズーム/ピンチ・PDF内リンク対応。
- **アプリ更新の通信**: 日常の起動では通信しない。**最後の確認から1日以上空いた起動時のみ**アプリ自体の更新を確認し、起動直後・未操作のときだけ適用する（設定から手動更新も可）。マニュアルの中身とは無関係の、同一オリジンのコード更新確認のみ。

## セキュリティ上の注意（重要）

このアプリは**動作中の外部通信がゼロ**（ネットワーク経由での流出はしない）です。ただし「漏洩しない」を成立させるには、以下の運用が前提になります。アプリだけでは守れない部分です。

- **端末にパスコード（ロック）を設定する。** 保存データ（IndexedDB）はアプリ側では暗号化しておらず、iOSの端末暗号化に依存する。パスコードが無い／ロック解除中の端末は中身を読める。**端末ロックが最後の砦。**
- **バックアップの扱いに注意。** エクスポートしたファイルは、暗号化しなければ中身は平文。**iCloud Drive・メール・共有PCなどに置くと、そこ経由で流出する**。端末ローカル（や手元のUSB等）だけに保管し、可能なら**パスワード暗号化**して保存する。
- **紛失・盗難＝物理的な流出。** オフラインでも端末を持たれれば意味がない（対策は端末ロック）。
- **アプリの配信URLを公開しても、外に出るのはアプリのコードだけ。** PDF・メモ・写真は端末内にのみ保存され、アップロードされることはない。ただし**マニュアル本体やバックアップを自分でどこかにアップしないこと**。
- **組織の資料を扱う場合は、私物端末に入れてよいかのルールを各自で確認する。** 技術的な流出は最小化できても、"保有そのものの可否" はアプリの外の問題。

アプリ内の「設定」タブ →「使い方と注意」からも、これらの注意を平易な言葉で確認できる。
